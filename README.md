# ProjModel для Jira для помощи в организации проектов

Полезный плагин для Jira со следующим функционалом:

**Анализ нагрузки сотрудников**

Можно посмотреть, какие сотрудники простаивают в плане количества задач, а какие наоборот ими перегружены. Панель с просмотром удобно подсвечивается ассоциативными цветами (например, красный - слишком большая нагрузка).

**ИИ-анализ проекта и задач**

Нейросеть анализирует сложность проекта и задачи и выдает личный взгляд на таск. При открытии окна с ИИ-анализом он сразу же запускается.

**Настройка ограничения области видимости у сотрудников разных ролей**

Внутри плагина тимлид может настраивать область видимости задач у сотрудников разных ролей: люди могут видеть задачи только своей роли. Работает по считыванию меток у задач и username пользователей Jira.

**Анализ дедлайнов по задачам**

Анализ дедлайнов задач: отсчет, сколько дней осталось на выполнение и вынос критически важных и срочных задач на первый план с соответствующей подсветкой.

*ProjModel пока доступен только для развертывания локально на устройстве/сервере и дальнейшей эксплуатации в рабочей/домашней сети.*

## **----------------- Инструкция по установке (для разработчиков) -----------------**

Для развертывания плагина и Jira локально необходимо иметь следующие инструменты:
- JDK 11
- IntelliJ IDEA
- Atlassian Plugin SDK 8.2.7
- Ollama 
- Qwen 2.5

**ОЧЕНЬ ВАЖНО**

1. Не храните проект:
- на флешке
- в OneDrive
- в папке с русскими буквами
- в слишком длинном пути
	
2. Нормальный путь для проекта:

***C:\dev\ProjModel-Plugin***
	
**Шаг 1. Клонировать проект**

Склонировать проект в подходящее место.
	
**Шаг 2. Установить JDK 11**

Нужен именно JDK 11, не JRE.
	
**Шаг 3. Настроить переменные среды Windows**
Нужно выставить:

JAVA_HOME = путь к JDK 11

Ну, например:

***C:\Program Files\Eclipse Adoptium\jdk-11.0.30.7-hotspot***
	
И также добавить в Path эту штуку:

***%JAVA_HOME%\bin***
	
Весь шаг 3 делается в утилите Пуск-> Изменение переменных среды. у Path эту штуку с java_home добавляйте в самый верх (перетащите потом)
	
**Шаг 4. Проверить Atlassian SDK**

В новом терминале выполнить: atlas-version
	
**Должно быть видно**:

- ATLAS Version: 8.2.7
- Java 11
- Путь к JDK, а не к ...\jre

**Шаг 5. Установить ИИ**

По ссылке https://ollama.com/download/windows для винды поставить установщик ollama, и, соответственно, саму Ollama

После установки через терминал прописать:
***ollama pull qwen2.5:3b***
	
**Шаг 6. Открыть проект в IntelliJ IDEA

Потом на всякий, после индексации, откройте в idea maven и нажмите reload all maven projects (sync->reload справа). шоб все работало
	
**Шаг 7. Запуск!**

Напишите в терминале idea

***atlas-run***

**Важно**:

- первый запуск может быть долгим
- команда не завершится сама, пока Jira работает
- это нормально

Она будет долго думать и говниться, но потом она выдаст вам ссылку на локально развернутую Jira и там уже вы должны будете войти в систему.

Логин и пароль для локальной жиры: admin admin.
	
Далее когда вы чет накодите, перезапускайте жиру.

1. Ctrl + C
2. atlas-clean
3. atlas-run
	
Ну и по новой.

## **Полезные команды**

Запуск Jira

***atlas-run***

Остановка Jira

***Ctrl + C***

Сборка плагина без запуска

***atlas-package***

Очистка сборки

***atlas-clean***
